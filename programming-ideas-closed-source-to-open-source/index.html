Programming Ideas: Closed Source to Open Source

Today I was trying to rack my brain and think of some big programs that don’t really have an open source equivilent. I posed the question to one of my co-workers, at first he mentioned a couple of games. I quickly told him to forget about games and focus on software.

I said “Imagine if GIMP didn’t exist as an open source alternative to Adobe Photoshop, that would be a good example.”

He pondered for a minute and he couldn’t really come up with anything.

Do all programs have an open source alternative?

After checking going through the list of programs I work with frequently I only found one, Whatpulse. If you don’t know what Whatpulse does, it tracks your mouseclicks and keyboard clicks and presents the data in a nice, reasonable fashion. It isn’t a keylogger, it just counts how many times keys and mouse buttons are pressed. Later on mouse movement distance was added, as well as scroll wheel distance.

This data is then pulsed to their servers so that things are available on the web. The site lets users have a dynamic image made so that people can display their stats. In fact, I have one on my About page.

One of the main reasons that the code isn’t open source is that if it was, people would figure out how to cheat (simulate key presses and mouse clicks), and it would make the competing aspect pointless. Some people don’t use Whatpulse because they’re worried about it actually being a keylogger (even though the software has been around for 10 years now and hasn’t had one reported incident).

But, let’s just assume that someone wants to run all of this on their local machine, and not pulse it to a webserver or anything else, I couldn’t really find an alternative for just tracking how many key presses and mouse clicks you have. This would be a great open source project for someone (maybe me) to start and pursue.

Damn, I’ve been using Whatpulse for 9 years?
